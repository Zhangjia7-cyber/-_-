我要基于Python ＋ streamlit 做一个元素人格测试仪的web app。我的具体需求如下：
1.它需要可以和用户进行交互，然后判断用户的人格类型
2.需要对项目进行 git 版本管理
3.后期会推送到github上 

需求描述的更新：元素人格测试仪 
1. 项目概述
开发一个基于 Streamlit 的交互式 Web 应用，通过 5 道选择题测试用户的人格倾向，将其归类为 火、水、土、风 四种元素之一（或混合元素），并展示对应的个性分析、生活建议以及可互动的动态形象。
2. 核心功能需求
2.1 测试流程
• 依次展示 5 道题目，每道题有 4 个选项，每个选项对应一种元素。
• 用户点击选项后，记录选择，自动进入下一题。
• 所有题目完成后，统计各元素被选次数，数量最多的元素即为用户人格类型（平局时判定为“混合元素”）。
• 支持重新测试，重置所有状态。
2.2 结果分析（缺点与建议）
为每种元素（包括混合）预定义：
• 优点：性格正面描述。
• 缺点：可能存在的不足。
• 生活建议：针对性的改进行动指南。
在结果页以突出卡片展示以上三项内容。
2.3 可互动的动态形象
• 结果页根据用户元素显示一个动态图标（例如跳动的 emoji）。
• 互动要求：
◦ 鼠标悬停时，图标放大，并出现一个文字气泡（如“🔥 燃烧吧！”）。
◦ 点击图标时，触发庆祝动画（如飘浮的气球、彩带效果）并显示随机友好提示。
• 混合元素显示多个图标并排，每个图标独立支持上述互动。
2.4 结果突出显示
结果页顶部用大号彩色发光文字明确写出“您的人格元素：火元素”等，确保用户第一眼看到。
2.5 题目选项居中布局
• 所有选项按钮在页面水平居中显示，按钮之间保留均匀间距。
• 适应移动端屏幕，按钮宽度适当缩放。
3. 视觉与交互增强需求
3.1 全局视觉风格
• 柔和的渐变背景或半透明背景图。
• 所有卡片使用圆角、阴影、半透明毛玻璃效果（backdrop-filter）。
• 四种元素介绍区块用各自对应的柔和底色 + 大图标（🔥💧🌍💨）。
• 页面整体动画：淡入、浮动、滑动等效果。
3.2 题目与选项趣味化
• 每个问题文本前加一个趣味 emoji（例如 🤔❓💡）。
• 每个选项文本前加一个与内容相关的 emoji（例如 ⚡ 直接面对、🤝 寻求帮助）。
• 题目文字可适当改为更生动表达，不改变原意。
3.3 互动细节
• 选项按钮 hover 时放大、变色、增加阴影。
• 点击选项时，通过 st.toast 短暂提示“已选择：xxx”。
• 进度条美观平滑（渐变色彩）。
• 重新测试按钮有点击波纹或颜色过渡效果。
3.4 其他装饰
• 页面顶部可添加动态 SVG 装饰或分隔线（或通过 CSS 动画实现等效效果）。
• 移动端响应式适配（卡片自适应宽度）。
4. 技术约束
• 使用 Streamlit 框架，代码为单一 .py 文件，可直接运行。
• 动态形象与交互效果通过 st.markdown 嵌入 HTML/CSS/JavaScript 实现，避免依赖外部网络资源（轻量级 CDN 除外）。
• 保持原有数据结构 ELEMENTS 和 QUESTIONS 可扩展性，但允许增加字段。
• 结果计算逻辑不变（统计最高频元素，平局为混合）。
5. 交付要求
• 输出完整可运行的 Python 代码。
• 代码需包含详细注释，尤其对 CSS 动画和 JS 交互部分。
• 最终效果需符合上述所有需求点。
